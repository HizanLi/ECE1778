# Assets Directory

This directory should contain the following image assets for the GeoClaim app:

## Required Assets

1. **icon.png** (1024x1024 px)
   - Main app icon displayed on device home screen
   - Should be a recognizable logo or symbol for GeoClaim
   - Suggested: Map pin or territory marker design

2. **adaptive-icon.png** (1024x1024 px) 
   - Android adaptive icon foreground layer
   - Should work well when masked to different shapes
   - Center 768x768px area is the safe zone

3. **splash-icon.png** (1242x2436 px)
   - iOS splash screen image
   - Displays while app is loading
   - Should match app branding

4. **favicon.png** (48x48 px)
   - Web version favicon
   - Small icon for browser tabs

5. **notification-icon.png** (96x96 px)
   - Android notification icon
   - Should be simple, monochrome design
   - Used in push notifications

## Creating Assets

You can use design tools like:
- Figma
- Adobe Illustrator/Photoshop
- Canva
- Or generate using AI tools like DALL-E, Midjourney

## Placeholder Assets

For development and testing, you can use solid color placeholders:
- Create simple PNG files with the app's primary color (#4CAF50)
- Add text or simple shapes to distinguish different assets

## Notes

- All images should be PNG format with transparency where appropriate
- Follow platform-specific guidelines for icon design
- Test assets on both iOS and Android devices
- Ensure high resolution for best quality on modern displays
